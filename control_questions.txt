	1. Що таке система контролю версій?
система що зберігає зміни і дозволяє повернутися до певної минулої версії
	2. Перерахуйте основні команди git
git init, git add..., git commit -m..., git log, git status
	3. Опишіть структру git проєкту та його складові
робоча директорія
область підготовених файлів
git-директорія
	4. При яких умовах файл стає відслідкованим і що відбуваєтся з git об’єктом при цьому?
за допомогою команди git add... файл стає відслідкованим, використавши цю команду ми створюємо для файлу blob-об'єкт
	5. Як перевірити статус каталогу файлів. Які стани характерні для git?
за допомогою команди git status. Стани: відслідкований (незмінений, змінений, підготовлений до комітів) та невідслідкований
	6. Що таке репозиторій. Які типи репозиторіів ви знаєте. Які сучасні репозиторії використовують git ?
Репрозиторій - каталог файлової системи, де є файли (по суті папка проекту)
типи репрозиторіїв - локальний та віддалений
сучасний репрозиторій - github
